## 데이터 타입
### 정수형 데이터 타입
| 타입           |정의| 범위                                                                             |UNSIGNED 사용가능 여부|
|--------------|---|--------------------------------------------------------------------------------|---|
| TINYINT(n)   |정수형 데이터 타입| - 1 Byte(2⁸) <br> - (-128~127) 또는 0~255 수 표현 가능                                |O
| SMALLINT(n)  |정수형 데이터 타입| - 2 Byte(2¹⁶) <br> - (-32,768~32,167) 또는 0~65536 수 표현 가능                       |O
| MEDIUMINT(n) |정수형 데이터 타입| - 3 Byte(2²⁴) <br> - (-8,388,608~8,388,607) 또는 0~16,777,215 수 표현 가능            |O
| INT(n)       |정수형 데이터 타입| - 4 Byte(2³²) <br> - (-2,147,483,648~2,147,483,647) 또는 0~4,294,967,295 수 표현 가능 |O
| BIGINT(n)    |정수형 데이터 타입| - 8 Byte(2⁶⁴) <br> - (2⁶⁴ - 1) 수 표현 가능 <br> - 무한 수 표현 가능이라고도 한다.               |O

### 소수형 데이터 타입
| 타입            | 정의                                                                                                                     | 범위                                     |UNSIGNED 사용가능 여부|
|---------------|------------------------------------------------------------------------------------------------------------------------|----------------------------------------|---|
| DECIMAL(m, d) | - 고정 소수형 데이터 타입고정(길이 + 1 byte) <br> - 화폐 데이터와 같이 데이터의 정확도를 요하는 경우에 주로 사용 <br> - M의 최대값은 65, D는 소수 자릿수이며 0이면 소수점 가지지 않음 | - 소수점을 사용한 형태 <br> - Default: m => 10  |X
| FLOAT(n)      | 부동 소수형 데이터 타입                                                                                                          | - 4 Byte(2^8) <br> - 부동 소수점을 사용한 형태    |X
| DOUBLE(n)     | 부동 소수형 데이터 타입                                                                                                          | - 8 Byte(2^8) <br> - DOUBLE을 문자열로 저장 |X

> ### 참고) UNSIGNED 속성
> - 해당 속성을 부여받은 컬럼은 음수값을 대입하지 못 한다.

### 문자형 데이터 타입
### Char
|타입| 정의                                                                                                                                         | 길이(단위 : byte) |
|---|--------------------------------------------------------------------------------------------------------------------------------------------|---------------|
|CHAR(n)| - 고정 길이 데이터 타입 <br> 지정된 길이보다 짧은 데이터 입력 시 나머지 길이는 공백으로 채워짐                                                                                  | 0 ~ 255       |
|VACHAR(n)| - 가변 길이 데이터 타입 <br> - 지정된 길이보다 짧은 데이터 입력 시 공백으로 채우지 않음 <br> - 저장 시 1-byte 혹은 2-byte 길이 Prefix 데이터를 저장. 이 Prefix 데이터는 값의 바이트 수에 대한 정보를 담는다. | 0 ~ 65,535    |

> ### 참고) Char 형태의 이진 데이터 타입
>  타입                       | 정의                 |길이(단위 : byte)|
--------------------------|--------------------|---|---|
> | - BINARY(n) <br> - BYTE(n) | CHAR 형태의 이진 데이터 타입 |최대 255|
> | VARBINARY(n)             | VARCHAR 형태의 이진 데이터 타입     |최대 65,535|

### TEXT
- 각 타입마다 비슷한 BLOB 타입과 같은 길이를 저장하지만 차이점은 저장 될 때 nonbinary string으로 저장 된다.

|타입| 정의                                             | 길이(단위 : byte)     |
|---|------------------------------------------------|-------------------|
|TINYTEXT(n)| - 문자열 데이터 타입 <br> - TINYBLOB과 같은 길이 값을 저장 가능   | 0 ~ 255           |
|TEXT(n)| - 문자열 데이터 타입 <br> - BLOB과 같은 길이 값을 저장 가능       | 0 ~ 65,535        |
|MEDIUMTEXT(n)| - 문자열 데이터 타입 <br> - MEDIUMBLOB과 같은 길이 값을 저장 가능 | 0 ~ 16,777,215    |
|LONGTEXT(n)| - 문자열 데이터 타입 <br> - LONGBLOB과 같은 길이 값을 저장 가능   | 0 ~ 4,294,967,295 |

> ### 참고) BLOB 이진 데이터 타입
> 타입|정의| 길이(단위 : byte)  |
> ---|---|----------------|
> TINYBLOB(n)|이진 데이터 타입| 최대 255         |
> BLOB(n)|이진 데이터 타입| 최대 65,535      |
> MEDIUMBLOB(n)|이진 데이터 타입| 최대 16,777,215  |
> LONGBLOB(n)|이진 데이터 타입| 최대 4,294,967,295 |
